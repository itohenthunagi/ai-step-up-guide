ブログ記事作成

あなたは[専門分野]の経験豊富なライターです。
[ターゲット読者]向けに「[記事タイトル]」について、
SEOを意識した[文字数]文字の記事を作成してください。

【記事の目的】
[記事を書く目的・読者に伝えたいこと]

【構成指示】
・導入（問題提起・読者の関心を引く）
・本文（3-5つのポイントに分けて詳細解説）
・まとめ（行動を促す呼びかけ）

【条件】
・見出し（H2、H3）を効果的に使用
・具体例や体験談を含める
・専門用語は分かりやすく説明
・読みやすい文章構成にする

問題解決提案

あなたは[業界/分野]の問題解決のスペシャリストです。
以下の課題について、実践的な解決策を提案してください。

【課題】
[具体的な問題・課題の内容]

【現状分析】
[現在の状況、制約条件、利用可能なリソース]

【求める提案内容】
1. 問題の根本原因分析
2. 解決策の選択肢（3-5案）
3. 各案のメリット・デメリット比較
4. 推奨案の詳細実行プラン
5. 成功指標と評価方法

【制約条件】
・予算：[予算範囲]
・期間：[実施期間]
・人員：[利用可能な人員]

アイデア発想

あなたは創造的思考の専門家です。
[目的・テーマ]について、革新的なアイデアを
[数量]個発想してください。

【背景・文脈】
[アイデアが必要な背景や状況]

【ターゲット】
[アイデアの対象となる人・市場]

【制約条件】
・実現可能性：[現実的 / 創造的 / 自由発想]
・予算規模：[低予算 / 中予算 / 予算制限なし]
・時間軸：[短期 / 中期 / 長期]

【アイデアの評価軸】
各アイデアについて以下で評価してください：
・独創性（1-5点）
・実現可能性（1-5点）
・インパクト（1-5点）
・コストパフォーマンス（1-5点）

学習コンテンツ作成

あなたは[専門分野]の教育のプロフェッショナルです。
[学習者レベル]の学習者向けに、「[学習テーマ]」について
分かりやすい学習コンテンツを作成してください。

【学習目標】
この学習を通して、学習者が[具体的にできるようになること]

【学習者の特徴】
・前提知識：[既に知っていること]
・学習動機：[なぜ学びたいのか]
・利用シーン：[どこで使うのか]

【コンテンツ構成】
1. 導入（なぜ重要なのか、学習のメリット）
2. 基本概念の説明（専門用語の定義）
3. 具体例・実例の紹介
4. 実践的な手順・方法
5. よくある間違いと対処法
6. 練習問題・確認テスト
7. 次のステップの案内

【条件】
・[文字数/時間]以内で完結
・図解や例が豊富
・段階的に理解を深める構成

効果的なメール作成

シチュエーション

重要なクライアントへのプロジェクト遅延のお詫びメール

プロンプト例

あなたは経験豊富なプロジェクトマネージャーです。
5年来の重要なクライアント（ABC社田中部長）に対して、
Webサイトリニューアルプロジェクトの納期が2週間遅れることを
誠実にお詫びし、信頼回復につながるメールを作成してください。

【状況詳細】
・遅延理由：追加要件により開発工数が増加
・元の納期：3月31日 → 新納期：4月14日
・今後の対応：毎週進捗報告、専任チーム追加

【メール要件】
・件名も含めて作成
・謝罪→理由説明→対応策→今後の関係継続意思の流れ
・ビジネスマナーに則った丁寧な文面
・400文字程度で簡潔に

期待できる結果

  • 適切な謝罪表現と責任の明確化
  • 具体的な対応策の提示
  • 信頼回復に向けた前向きな姿勢

企画書作成支援

シチュエーション

社内向け新サービス提案の企画書作成

プロンプト例

あなたは戦略コンサルタントです。
IT企業の新規事業として「中小企業向けDX支援サービス」の
企画書を作成してください。

【会社情報】
・従業員200名のシステム開発会社
・既存事業：大企業向けシステム開発
・新規事業予算：年間3000万円

【サービス概要】
・ターゲット：従業員20-100名の製造業
・内容：業務デジタル化コンサル+システム導入
・価格帯：月額20-50万円

【企画書構成】
1. 事業機会（市場分析、競合状況）
2. サービス詳細（内容、差別化ポイント）
3. 事業計画（売上予測、必要リソース）
4. リスクと対策
5. 実行スケジュール

各セクション300-500文字で、データや根拠も含めて作成してください。

期待できる結果

  • 論理的で説得力のある企画書構成
  • 市場データに基づいた客観的分析
  • 実現可能性の高い事業計画
